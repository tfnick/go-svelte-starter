Features

A practical SaaS foundation with marketing pages in the same executable.

The public site stays server-rendered for crawlers, while the authenticated workspace keeps the Svelte workflow users expect.

01

Server-rendered landing pages

Native Go templates keep the public surface fast, inspectable, and easy to replace without changing the dashboard build.

02

Embedded Svelte workspace

Vite builds the SaaS UI into frontend/dist, and Go embeds it so production deployment remains a single binary.

03

Integrated checkout continuation

Visitors pick a plan on the marketing page, authenticate inside /app, and continue through the existing order and payment APIs.

04

Hard API boundary

/api remains an API-only namespace, so missing API routes return API errors instead of accidentally serving frontend HTML.
